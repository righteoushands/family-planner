  show them read-only with NO "Change" button. The write loop applies to

      "Change" button carrying the same data-date/data-slot.

  Keep all existing G1b-1 display (recipe state labels, source tag, off-shopping

STEP 3 — render_meal_wizard_step4.py, the JS (concatenated string literals, like
  s3Save; NOT an f-string; '\\n' for any JS newline; no nested quotes pulled
  into the Python f-string):
    window.s4Keep = function(date, slot){
      - read the name/ingredients/protein inputs for that date::slot via the
        data-attributes; if name is empty, show an inline message and return
        (do not POST an empty meal).
      - build payload = { date: date, slot: slot, name: <name>, source: "manual",
        ingredients: <ingredients or "">, protein: <protein or "">,
        recipe_id: "", recipe_on_request: true };   // recipe default
      - fetch('/meal-wizard-step4-confirm', {method:'POST',
        headers:{'Content-Type':'application/json'}, body: JSON.stringify(payload)})
        .then(r=>r.json()).then(j=>{ if(j&&j.ok){ window.location.href =
        '/meal-wizard-step4'; } else { <inline error> } }).catch(<inline error>);
    };
    window.s4Change = function(date, slot){
      - payload = { date: date, slot: slot };
      - fetch('/meal-wizard-step4-remove', {method:'POST',
        headers:{'Content-Type':'application/json'}, body: JSON.stringify(payload)})
        .then(r=>r.json()).then(j=>{ if(j&&j.ok){ window.location.href =
        '/meal-wizard-step4'; } else { <inline error> } }).catch(<inline error>);
    };
  Wire the buttons with onclick="s4Keep('YYYY-MM-DD','dinner')" /
  onclick="s4Change('YYYY-MM-DD','dinner')". Reload-on-success keeps the page in
  sync with the session with no client-side state to drift.
