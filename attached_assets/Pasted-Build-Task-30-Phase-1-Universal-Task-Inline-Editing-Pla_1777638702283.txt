Build Task #30 — Phase 1: Universal Task Inline Editing. Plan before building. What and why: Tasks currently have no edit path after creation. Done and Archive are the only actions on the /tasks page, and Step 5 on /mom has no edit controls at all. This task adds inline editing to both surfaces and fixes the fragile index-based identification on the existing mutation routes. Done looks like: Every task row on the /tasks page and in Step 5 has an Edit button that expands an inline panel below the row. The panel contains a text field for the task name, a date picker for due date, a person selector for assigned_to, a priority selector, and for recurring tasks a recurrence summary with an Edit Recurrence link that opens the existing recurrence fields. A Save button POSTs to the new /task-update route and collapses the panel. A Mark Done button marks the task complete. The inline panel uses the same visual pattern as the plan importer item cards. New route /task-update in app.py: Accepts POST with id, text, assigned_to, due_date, priority, and optionally recurring fields matching the /add-task schema. Finds the task by id not index. Updates only the fields provided. Saves with safe_save_json via save_manual_tasks. Returns JSON ok:true. Must acquire _MANUAL_TASKS_LOCK. Register in _JSON_PATHS. Fix /task-done and /task-delete to use id instead of index: Both routes currently use integer index which is fragile. Change both to find the task by id first, fall back to index only if id is not provided for backward compatibility with any existing callers. Use _MANUAL_TASKS_LOCK on both. Step 5 changes in render_misc.py: Add an Edit button to every row in all three sections. Clicking expands an inline panel with due date picker, person selector, priority selector, and Save and Mark Done buttons. Save POSTs to /task-update via fetch. Mark Done POSTs to /task-done via fetch. Both collapse the panel on success and refresh the Step 5 fragment by re-fetching /mom-step/tasks. The inline panel toggle is handled by a small JS block at the bottom of the _render_tasks_step return value — no global functions, inline script only, matching the existing pattern in that function. /tasks page changes in render_misc.py: Replace the existing Done and Archive buttons with three buttons: Edit expands inline panel, Done, Archive. The inline panel contains all editable fields including full recurrence editor matching the /add-task form fields. Save POSTs to /task-update. Done POSTs to /task-done. Both use fetch and refresh the page on success. The panel uses the existing card styling. Constraints: Edit only render_misc.py and app.py. No changes to render_schedule.py, render_plan_importer.py, or any other file. CSS for the inline panel goes inline in the rendered HTML using existing CSS variables. All new POST routes use id not index. py_compile must pass on both files. Plan before building.
