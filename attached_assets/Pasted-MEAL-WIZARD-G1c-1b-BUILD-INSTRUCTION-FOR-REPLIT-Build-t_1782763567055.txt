BUILD STEP 1 — render_meal_wizard_gen.py: build_wizard_meal_prompt
================================================================
    def build_wizard_meal_prompt(session: dict, target_keys: list) -> str:
        """Assemble the one-pass generation prompt from the wizard session.
        Returns a single string. Built as a list of lines joined with newline
        (Rule 1/2/7 — no f-string backslash/quote hazards). Family-agnostic:
        any family facts come from app_settings / meal rules, never hardcoded
        (Rule 19)."""

Inputs to pull (all fail-soft; default to "" / [] when missing):
  - dates+slots: from target_keys — group into a human list "YYYY-MM-DD — slot".
  - inventory: session.get("confirmed_inventory","")  (what Lauren entered in the
    wizard — NOT the legacy _get_inventory).
  - use_soon: session.get("use_soon_items","").
  - used_proteins: session.get("used_proteins",[]).
  - complexity: session.get("confirmed_complexity","normal").
  - confirmed_meals: session.get("confirmed_meals",{}) — list the ones whose date
    falls in this window so the model can plan a COHERENT week around them
    (it must NOT fill or change them).
  - meal rules / dietary: reuse render_lorenzo._get_meal_constraints() if it is
    importable at module level; if importing causes a circular import, replicate
    its minimal read of data/meal_rules.json. (Family dietary facts live here and
    in app_settings — Rule 19.)
  - calendar: reuse render_lorenzo._get_calendar_this_week(<window start iso>)
    for busy-night context (already merged/deduped per the diagnosis). Same
    circular-import fallback rule.
  - saved recipes: optional — reuse render_lorenzo._get_saved_recipes() so the
    model can prefer dishes Lauren already has recipes for. Optional context.

Prompt content (assemble these as lines; condensed from Lauren's 5/31 Lorenzo-
fixes corrections + item 44 + protein rotation):
  - Role: a Catholic homeschool family's meal planner producing a DRAFT week the
    mother will review and edit.
  - "Fill EXACTLY these meal cells, and ONLY these:" + the grouped date—slot list.
    "Do not add any day or meal type that is not in this list." (Correction 5 /
    Rule 16 — only the categories Lauren chose.)
  - "These meals are ALREADY decided by Lauren — do NOT propose or change them;
    plan a coherent week around them:" + the in-window confirmed list (or "none
    yet"). (Correction 4.)
  - "USE-SOON items — each MUST be used in at least one meal this week:" +
    use_soon. "Do not ignore them." (Correction 2 / Issue 13 — the ignored
    bananas.)
  - "On-hand inventory (note the FORM — fresh / canned / frozen / dried):" +
    inventory. "Respect the form exactly: if an item is canned, do not plan a
    dish that needs it fresh, and vice versa; treat the canned/frozen form as
    usable as-is." (Item 44.)
  - "Do NOT repeat a main protein already used this week:" + used_proteins +
    "and rotate proteins across the days you plan (no protein twice unless
    unavoidable)."
  - "Standing meal rules — follow exactly, including any meatless-Friday /
    liturgical rules:" + the rules string.
  - "Calendar for the week (keep meals quick on busy days):" + the calendar
    string.
  - "Effort level for this week: " + complexity + " — match it."
  - IF any target slot is dad_lunch: "John's (Dad's) lunch must NEVER be the same
    as that day's family dinner; he prefers meat over salad or rice." (Correction
    9.)
  - "Suggest real, makeable dishes. You may include meals that need a few
    shopping items, but never claim an ingredient is on hand unless it is in the
    inventory above." (Anti-fabrication — the diseases.)
  - OUTPUT CONTRACT (must match G1c-1a's parser): "Return ONLY a JSON object, no
    prose before or after, in EXACTLY this shape:"
        {"meals": {"YYYY-MM-DD": {"slot": {"name": "...", "protein": "...",
         "ingredients": "...", "note": "..."}}}}
    "Include only the date/slot cells listed above. protein = the single main
    protein, or empty string if meatless. ingredients = brief comma-separated
    list. note = optional, <= 8 words."

    elif path == "/meal-wizard-generate":
        # read/discard any body (in _JSON_PATHS)
        _g_cl = int(self.headers.get("Content-Length","0") or 0)
        if _g_cl:
            try: self.rfile.read(_g_cl)
            except Exception: pass
        try:
            _g_session = load_meal_wizard_session()
            _g_targets = wizard_target_slot_keys(_g_session)
            if not _g_targets:
                _g_result = {"ok": True, "generated": 0, "target": 0,
                             "message": "No empty slots to generate."}
            else:
                _g_prompt   = build_wizard_meal_prompt(_g_session, _g_targets)
                _g_settings = load_app_settings()
                _g_key = (_g_settings.get("anthropic_api_key","")
                          or _g_settings.get("family_constraints",{}).get("anthropic_api_key",""))
                if not _g_key:
                    _g_result = {"error": "No API key set in Settings"}
                else:
                    _g_resp = requests.post(
                        "https://api.anthropic.com/v1/messages",
                        headers={"x-api-key": _g_key,
                                 "anthropic-version": "2023-06-01",
                                 "content-type": "application/json"},
                        json={"model": <SAME MODEL STRING /meal-generate USES>,
                              "max_tokens": 4096,
                              "messages": [{"role":"user","content": _g_prompt}]},
                        timeout=90)
                    _g_resp.raise_for_status()
                    _g_text = "".join(b.get("text","") for b in _g_resp.json().get("content",[])
                                      if b.get("type") == "text")
                    _g_suggestions = parse_wizard_meal_response(_g_text, _g_targets)
                    update_meal_wizard_session({"suggested_meals": _g_suggestions})
                    _g_result = {"ok": True, "generated": len(_g_suggestions),
                                 "target": len(_g_targets),
                                 "suggested_meals": _g_suggestions}
        except Exception as _ge:
            _g_result = {"error": str(_ge)}
        self.send_response(200); self.send_header("Content-Type","application/json"); self.end_headers()
        try: self.wfile.write(json.dumps(_g_result).encode())
        except BrokenPipeError: pass
        return
